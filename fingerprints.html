Fingerprints

A secure hypertext transfer protocol (HTTPS) connection, known as secure sockets layer (SSL) aims to both protect the user of a website from being tracked and protect the host of the website from being hacked by requiring the website to send a certificate to a certificate authority (CA), proving the website's authenticity. An HTTPS proxy appliance, however, can interrupt a "secure" connection to a user and can emulate the website to the user while also tracking them by presenting a fake certificate to the user. This is an example of a man-in-the-middle attack, also called MITM, which threatens the security of both the host and user.

The certificate proving a website is secure is identified by a hash, a group of two digit hexadecimal numbers, which is called the website's fingerprint. Hashes are considered "better" when based upon Secure Hashing Algorithm 1 (SHA1) compared to Message Digest 5 (MD5) and when they are longer due to the chance that more digits would change, making it easier to detect when someone makes a fake certificate (i.e. an HTTPS proxy appliance). To detect an SSL interception, simply compare your browser's fingerprint with the actual fingerprint of the website (preferrably by checking the fingerprint here). The only way to prevent an interception, however, is to use a secure network that you know is safe, such as your house wifi.

False positives to interception checks can occur due to many websites having multiple certificates, and not user recieves the same one, so fingerprints might not match due to the wrong certificate being checked.
False negatives can occur when the intercepter has knowledge of the check, and may create an exception to the website being checked, so that website may be the only one that displays the authentic certificate. A false negative can also occur when the proxy appliance edits the website checker to display the forged thumbprint so it looks as if the connection is secure.
